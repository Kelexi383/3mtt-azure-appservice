Azure App Service Web App Deployment
- Lab Summary
Course/Program: 3MTT Cloud Engineering Learning Program
Focus Area: Platform as a Service (PaaS) Application Delivery

1. Project Overview & Architecture
This project demonstrates modern cloud-native application delivery using Platform as a Service
(PaaS) via Microsoft Azure App Service. By deploying a headless Node.js web application on a
fully managed infrastructure environment, this project highlights how PaaS abstractions reduce
operational and infrastructure overhead compared to traditional Infrastructure as a Service
(IaaS) setups.
Environment Details:
● Language Runtime Stack: Node.js 22 LTS
● Hosting Platform: Azure App Service (Linux)
● Pricing Tier / SKU: Free (F1) Pricing Plan
● Deployment Region: West US 3
● Deployment Pipeline: Continuous Integration / Continuous Deployment (CI/CD) via
GitHub Actions

2. How the Automated Deployment Works
The application architecture utilizes an automated Git-based DevOps pipeline instead of manual
file uploads:
1. The Source: The codebase resides within a public GitHub repository hosting the
application infrastructure configs (package.json) and application server logic (server.js).
2. The Bridge: Linking the repository via the Azure Deployment Center automatically injects
a cryptographically secure GitHub Actions Workflow file into the repository
(.github/workflows).
3. The Trigger: Every time a code modification or update is committed to the main branch,
GitHub automatically triggers a cloud runner to build, bundle, and securely push the
updated package to the Azure App Service container hosting environment over HTTPS.

3. Verification and Operational Features
The deployment was successfully audited and verified through the following infrastructure

features:
● Global Availability: The web app is accessible worldwide over a secure, encrypted
HTTPS endpoint generated by Azure's default routing architecture.
● Configuration Management: Evaluated the "Configuration" blade within the Azure Portal
to manage application settings and environmental variables dynamically without modifying
application code directly.
● Operational Scaling: Audited metrics dashboards for vertical scaling (moving up pricing
tiers for memory/CPU specs) and horizontal scaling (adding multiple running app
instances).
